What Ase2UE Does

Ase2UE is an editor tool for artists and solo developers using Aseprite and Paper2D. It turns a saved .ase or .aseprite file into Unreal assets.

Texture Import

Runs the local Aseprite exporter and imports the generated spritesheet PNG into Unreal.

PaperSprites

Creates or updates PaperSprite assets from Grid cells or Aseprite JSON frame data.

PaperFlipbooks

Creates flipbooks from Aseprite frame tags such as Idle, Run, Attack, or Jump.

Sprite Sockets

Reads 1x1 marker pixels from named layers and writes sockets onto generated sprites.

Stable Reimport

Updates existing generated assets in place, so iteration does not create duplicate piles.

Optional Hitbox Data

Can create an advanced Blueprint-readable data asset for HIT, HURT, and SOCKET data.

Requirements

  • Unreal Engine 5 project.
  • Paper2D plugin enabled.
  • Aseprite installed locally.
  • A saved .ase or .aseprite file.

Ase2UE creates assets in the editor. It is not a runtime importer.

Quick Start

  1. Enable the Ase2UE plugin.
  2. Restart Unreal if prompted.
  3. Enable Paper2D if it is not already enabled.
  4. Open Project Settings > Plugins > Ase2UE.
  5. Set Aseprite Executable Path if needed.
  6. Create an Ase2 Source asset in the Content Browser.
  7. Set Aseprite File to your .ase or .aseprite file.
  8. Set Import Destination to a folder under /Game.
  9. Choose Grid or Auto extraction.
  10. Click Import Now.

How Each Feature Works

Import Now

Import Now exports the Aseprite file, imports the generated PNG, reads the JSON data, and updates Unreal assets in the selected destination folder.

Generated Assets

A file named Player.aseprite creates a stable layout like this:

/Game/YourFolder/Player/
  Textures/T_Player_Sheet
  Sprites/SP_Player_0000
  Flipbooks/FB_Player_Idle
  Data/DA_Player_Hitboxes

The Data folder appears only when advanced hitbox data generation is enabled.

Reimport

Edit in Aseprite, save, then click Import Now again. Ase2UE updates the same generated assets instead of making duplicate copies.

Ase2 Source Settings

Aseprite File
The source .ase or .aseprite file.
Import Destination
The Unreal content folder where generated assets are created. Use a folder under /Game.
Pivot Mode
The PaperSprite pivot mode used by generated sprites. Center Center and Bottom Center are common choices.
Default Material
The Paper2D material applied to generated sprites.
Create Flipbooks From Tags
Creates one PaperFlipbook for each Aseprite frame tag.
Import Marker Layers
Reads layers named with SOCKET, HIT, or HURT prefixes.
Generate Hitbox Data Asset
Optional advanced setting for Blueprint-readable marker and slice data.

Grid vs Auto From Aseprite JSON

Grid

Grid mode cuts the spritesheet into fixed-size cells. Use it when each animation frame uses the same canvas size, especially for socket marker workflows. 

Sprite Extraction: Grid
Grid Cell Width: 80
Grid Cell Height: 80

Auto From Aseprite JSON

Auto mode reads frame rectangles from Aseprite's exported JSON. Use it for packed or trimmed frames when exact socket marker placement is not needed.

Flipbooks From Tags

Aseprite tags become PaperFlipbooks.

Idle -> FB_Player_Idle
Run -> FB_Player_Run
Attack -> FB_Player_Attack

Enable Use Aseprite Frame Durations if you want the flipbook timing to match the timing set in Aseprite.

Marker Layers And Sockets

Marker layers are separate Aseprite layers used for gameplay helper points. Ase2UE hides these layers from the main texture, exports them separately, and writes matching sockets onto generated PaperSprites.

Create A Socket

  1. Create a new layer in Aseprite.
  2. Name it with a supported prefix.
  3. Place one visible 1x1 pixel where the socket should be.
  4. Save the file.
  5. Click Import Now in Unreal.
SOCKET_Hand
SOCKET_WeaponTip
HIT_WeaponStart
HIT_WeaponTip
HURT_Body

Does Marker Color Matter?

No. Ase2UE reads visible non-transparent pixels. The color is only for your own organization.

Weapon Trace Example

For a melee weapon, use two marker layers:

HIT_WeaponStart
HIT_WeaponTip

After import, use those socket locations as the trace start and trace end.

Import Report

After import, Ase2UE shows a report with separate sections:

  • What was done: successful actions, such as imported markers or applied sockets.
  • Notes: helpful information that is not a problem.
  • Warnings: items that need attention.

Troubleshooting

Aseprite executable was not found

Set Aseprite Executable Path in Project Settings > Plugins > Ase2UE.

The file picker does not show my file

Switch the picker to all files or paste the full path.

Sockets do not appear

  • Enable Import Marker Layers.
  • Use a layer name starting with SOCKET, HIT, or HURT.
  • Use a visible, non-transparent 1x1 pixel.
  • Use Grid mode for exact canvas placement.

Socket position looks offset

PaperSprite sockets are stored in pivot-space, not Aseprite canvas coordinates.

Aseprite pixel: 29, 40
Pivot on 80x80 sprite: 40, 40
PaperSprite socket transform: X=-11, Y=0, Z=0

Flipbooks were not created

Make sure Create Flipbooks From Tags is enabled and the Aseprite file has frame tags.
