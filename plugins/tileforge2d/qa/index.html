TileForge2D QA Checklist

A release-readiness tracker for TileForge2D while editor, runtime, procedural systems, sample content, documentation, and marketplace prep are hardened.

Back to TileForge2D Repo Docs

Core QA

  • Create tileset and map assets.
  • Paint, erase, fill, line, box, pick, undo, and redo.
  • Lock, hide, add, and remove layers.
  • Place a runtime map actor and confirm tiles render.
  • Confirm collision blocks a test character.

Runtime QA

  • Destroy a tile at runtime.
  • Place a tile at runtime.
  • Interact with a tile or Tile Actor.
  • Capture and apply runtime save data.
  • Generate and clear biome/dungeon test data.
